## Решения:

Сегодня занимался в основном логикой `CodeEngineService`. Он написан с помощью ИИ, так как этот сервис имеет довольно сложную для меня логику, но попробую объяснить его механику так, как понял сам: всё работает благодаря **WebWorker**.

Сервис имеет главный метод `execute(code: string, tests: string)`, который принимает исполняемый код и тесты к нему, а возвращает `Promise`.
Внутри создается код воркера, в котором выполняется **Monkey Patching** для `console.log`. Теперь он вместо вывода в консоль браузера (DevTools) отправляет логи наверх к сервису. В будущем планирую сделать так, чтобы эти логи прокидывались в наш встроенный терминал через `Emitter`.

В этом же коде воркера используется конструкция `try...catch`. Сначала выполняется основной код, а затем тесты (которые выглядят как `if (...) throw new Error(...)`). Если что-то падает с ошибкой, `catch` это ловит и передает в сервис.

Сам сервис создает новый `Blob`, который принимает код воркера и создает в оперативной памяти изолированный файл. Его и использует `Worker`. Дальше сервис просто слушает сообщения об ошибках, логах или успешном выполнении и, в зависимости от ответа, делает `resolve` или `reject` промиса. Также предусмотрен таймер для защиты от бесконечных циклов. Попытался объяснить понятно, хотя таким навыком не владею. Также хочу отметить, что JSDoc для этого метода я написал сам, поэтому он кривой:

```typescript
/**
 * The method executes code and runs tests in an isolated sandbox,
 * where all console.log statements are output via an emitter.
 * In case of an error, it returns a
 * Promise with a reject status containing the error.
 *
 * @param {string} code - executable code
 * @param {string} tests - code with tests such as `if (...) throw new Error(...)`
 *
 * @throws {Error} In case of an error in the executable code or failure of tests
 *
 * @example
 * const code = `function sum(a, b) {
 * return a + b;
 * }`
 *
 * const tests = `
 * if (typeof sum !== 'function') throw new Error("Функция sum не найдена. Доступ запрещен.");
 * if (sum(2, 3) !== 5) throw new Error("Сигнатура sum(2, 3) не совпадает с ожидаемой (5).");
 * if (sum(-1, 1) !== 0) throw new Error("Сигнатура sum(-1, 1) не совпадает с ожидаемой (0).");
 *`
 * await execute(code, tests);
 */
```

Пропсы для `CodeEditor` я реализовал еще вчера, но сил на запись в дневник уже не осталось. С их помощью теперь можно адаптировать компонент под разные нужды: для лендинга (визуал), квиза или виджета по типу CodeWars.

## Проблемы:

Изначально планировал делать `CodeEngineService` через отдельный файл `worker.ts`, запуская код через `new Function(code)` внутри воркера. Однако eslint выдавал ошибку. Я хотел отключить правило через комментарий `// eslint-disable...`, но вспомнил, что сам же ранее установил в конфиге запрет отключение правил с помощью комментариев.

![alt text](https://media1.tenor.com/m/LX8vpjBcq_sAAAAC/dersayder228.gif)

В итоге решил не изменять конфиги и после беседы с ии перешел на метод с `Blob` и полной изоляцией кода. У этого способа больше плюсов: после исполнения песочница полностью удаляется. В старом варианте воркер переиспользовался, из-за чего переменные, объявленные через `var`, могли оставаться в памяти. Например, если дважды запустить код `var aboba = 123`, могли возникнуть ошибки из-за повторного объявления имен.
