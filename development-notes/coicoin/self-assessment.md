## 2. Описание проделанной работы

В рамках проекта я полностью разработал и реализовал следующие части приложения:

### Роутер

- Реализован SPA роутер с использованием `historyAPI`.
- Работа с параметрами, `queries`, навигация по страницам, безопасность страниц, редиректы.
- Расширение функционала: учёт `queries` при сравнении путей, переходы по `id`.
- Основные сложности: корректные переходы с сохранением состояния и параметров.

### CSS / SCSS

- Константы вынесены в глобальные CSS-переменные, чтобы не импортировать их в каждый модуль SCSS.
- Адаптивная верстка и корректная интеграция стилей для всех страниц.
- Запомнил что import устарел и начал использовать use.
- Использовал mixins.

### DevOps / CI/CD

- Настройка автоматического деплоя через GitHub Actions.

### Архитектура

- Разделение проекта на компоненты и слои (`DTO`, `Repository`, `Mapper`) для изоляции бизнес-логики.
- Применение паттернов `Singleton`, `Observer`.
- Выделение слоя API для взаимодействия с внешними сервисами.
- Создание архитектуры SPA с поддержкой динамических маршрутов, состояния и параметров.

---

### Landing Page

- Создание общей структуры страницы: `Header`, `Footer`, основной контент.
- Кнопки: `Login`, `Logout`, `Register`, `Main`, `About`, `Start Practice`.
- Реализация базовой интерактивности и адаптивной верстки.

### About Page

- Страница с `Header` и `Footer`.
- Секция `Contributors` с анимированными карточками (avatar, роль, имя, GitHub, вклад).
- Интеграция GitHub API для получения аватаров.
- Кастомные стили и анимации при наведении.

### Quiz Widget Page

- SPA-страница с собственным `Header` и `Footer`.
- Вопросы с кодовыми фрагментами и вариантами ответов c анимацией.
- Кнопки `Skip` и `Continue`, прогресс-бар, уровни, таймер.

### TrueFalse Widget Page

- Таймер, прогресс-бар и уровень.

## 3. Два личных Feature Component (для защиты)

Хочу выделить 2 виджета:

1. **Quiz Widget Page** – самый сложный компонент из-за сочетания нескольких интерактивных элементов:
   - Собственный `Header` и `Footer`.
   - Вопросы с несколькими вариантами ответа и `code snippet`.
   - Анимации для вариантов ответа и кнопок `Skip` / `Continue`.
   - Timer меняющий цвет при достижении лимита, уровень (level), прогресс-бар.
   - Работа с состоянием SPA через собственный Router с поддержкой `queries`, `params`, `location` и `navigate`.
   - Создание общих компонентов: `Progress Bar`, `Timer`, `Inline Code Text`, `Score Component`.
   - Деление проекта на компоненты.
   - Хранение данных в State.
   - Взаимодействие с Supabase.
     Этот компонент демонстрирует владение TypeScript + Vanilla JS, сложной логикой UI и архитектурным подходом.

2. **TrueFalse Widget Page** – сложный компонент из-за высокой интерактивности и визуального представления:
   - Использование уже разработанного прогресс-бара, похожий но новый progress, доработанный общий timer.
   - Таймер доработан с возможностью настройки последовательного или реверсивного отсчета, и добавление разделетеля, .
   - Анимации кнопок `True` / `False` и реакций на действия пользователя.
   - Взаимодействие с общим `Score Component` для отображения результатов.
   - Деление проекта на компоненты.
   - Хранение данных в State.
     Этот компонент показывает навыки в управлении состоянием, динамическом рендеринге и построении интерактивного UX.

В обоих из них был внедрен **State**

- Создан глобальный `store` для хранения текущего состояния викторины, и исключения сильной связки через props.
- Использован паттерн `Observer` для подписки UI-компонентов на изменения состояния (`state changes`) и автоматического обновления интерфейса.
